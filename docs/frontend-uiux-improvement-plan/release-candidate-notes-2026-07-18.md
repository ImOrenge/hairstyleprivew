# HairFit UI/UX 출시 후보 안내 — 2026-07-18

## 고객에게 달라지는 점

- 헤어스타일 생성 접수 후 화면을 닫아도 서버 작업은 계속 진행되며, 마이페이지 `작업 현황`에서 대기·진행·완료·실패를 다시 확인할 수 있습니다.
- 생성이 끝나면 인앱 상태와 이메일로 알려드리고, 링크를 열면 로그인 후 같은 생성 결과로 돌아갑니다.
- 홈의 단순 생성 기록은 실제로 시술하기로 확정한 스타일 카드로 바뀝니다. 선택한 스타일 이미지와 관리 가이드를 바로 확인할 수 있습니다.
- 생성 결과에서는 스타일 선택과 시술 확정의 의미를 분리했습니다. 시술 확정 뒤에는 해당 스타일이 잠기며 다른 스타일은 새로 생성해야 합니다.
- 패션 룩북도 접수 뒤 백그라운드에서 처리되고 완료 상태·실패 환불·재시도 경로를 제공합니다.
- 결제 직전 크레딧 차감량, 현재 잔액, 차감 후 잔액, 결과 보관기간을 다시 확인합니다.

## 개인정보와 보관

- 생성에 사용한 원본 사진은 모든 결과가 완료되면 정리하고, 일부 실패 또는 전체 실패 때만 무료 재시도를 위해 접수 후 최대 24시간 보관합니다.
- 완료 이메일의 수신자 정보와 본문 payload는 발송 완료 후 30일, 운영 판정이 필요한 실패·불확실 발송은 90일 뒤 비식별화합니다.
- 계정 삭제는 앱 데이터와 사진을 먼저 삭제한 뒤 로그인 계정을 마지막에 삭제하며, 중단돼도 같은 요청을 안전하게 재개합니다.
- 확정 스타일 카드에는 실제로 선택·확정한 결과 이미지만 표시하고 임의의 대체 이미지를 확정 결과처럼 보여주지 않습니다.

## 출시 시 기본 비활성 기능

- 모바일 Push는 EAS/APNs/FCM 자격 증명과 iOS/Android 실기기 수신 검증 전까지 꺼져 있습니다. 그동안 인앱 작업 현황과 완료 이메일이 기본 알림입니다.
- PortOne 운영 checkout은 sandbox·webhook·환불 검증 전까지 `waitlist` 모드를 유지할 수 있습니다.
- 운영자가 장애 대응으로 새 접수를 일시 중단해도 이미 접수된 헤어 생성과 패션 룩북, 상태 조회, 정산·환불, 완료 알림 처리는 계속됩니다.

## 출시 전 남은 증거

- 운영 Supabase migration·cron과 Cloudflare App/Workflow coordinated rollout
- 실제 Resend 수신 1회·중복 0회와 종료 후 재진입
- PortOne sandbox 결제·중복 webhook·환불 receipt
- iOS/Android Push, app-link, 200% 글자, VoiceOver/TalkBack 실기기 검증
- GitHub-hosted CI 첫 green run과 branch protection required check 등록

위 증거가 완료되기 전에는 이 문서를 “운영 출시 완료” 공지로 사용하지 않는다.
