# 사용자 행동 문구 계약

이 문서는 HairFit 웹·앱에서 같은 행동을 같은 말로 안내하기 위한 Phase 12B 용어 사전이다. 코드의 enum, API field, provider 이름은 내부 식별자이며 사용자 행동 문구로 그대로 노출하지 않는다.

## 핵심 용어

| 용어 | 사용자 의미 | 사용하는 위치 | 피해야 할 표현 |
| --- | --- | --- | --- |
| 선택 | 생성된 여러 결과 중 현재 사용할 후보를 고르는 되돌릴 수 있는 행동 | 결과 카드, 비교, 패션 추천 진입 | 선택만으로 `확정`, `잠금`되었다고 표현 |
| 시술 계획 확정 | 선택한 결과로 시술 기록을 만들고 이후 후보 변경을 막는 되돌릴 수 없는 행동 | 최종 확인 CTA와 확인 dialog | 일반 카드 선택 CTA를 `확정`으로 표현 |
| 변경 불가 | 확정 뒤 선택이 잠긴 결과를 사용자가 이해할 수 있게 설명하는 상태 | 확정된 결과 안내 | `selection lock`, `잠금 플래그` 같은 내부 상태명 |
| 비용 | 작업 실행 전에 최신 잔액과 함께 확인하는 크레딧 사용량 | 생성·재생성·Styler 견적 | 실행 뒤에만 차감을 알리거나 `reservation` 원문 노출 |
| 재시도 | 같은 작업에서 실패한 단계만 다시 수행하는 복구 행동 | 실패 후보, 네트워크 조회 | 새 비용이 발생하는 새 작업을 `재시도`로 표현 |
| 다시 생성 | 현재 결과를 보존하고 최신 비용 확인 뒤 새 작업을 접수하는 행동 | 결과 더보기, 재생성 확인 dialog | 무료 복구처럼 보이는 `재시도` |
| 결제 후 복귀 | 충전 뒤 원래 작업으로 돌아오되 자동 실행하지 않고 최신 비용을 다시 확인하는 흐름 | 부족 잔액 안내, 결제 복구 | `returnTo`, callback, webhook 같은 구현 용어 |

## 상태·오류 문구

- 결제 provider·API·webhook·base64·payload·enum 원문은 고객 화면에 표시하지 않는다.
- 결제 실패는 `카드 확인`, `결제 상태 다시 확인`, `고객지원`처럼 사용자가 취할 수 있는 다음 행동으로 안내한다.
- 운영자 화면도 요청 중복 방지 의미를 `결제 취소 요청을 다시 보내지 않습니다`처럼 행동 단위로 설명한다.
- 알 수 없는 plan/status/provider code는 원문 대신 `정보 확인 필요` 또는 `상태 확인 중`으로 표시한다.

## 검증 범위

`npm run copy-terminology:contract:test`는 다음을 고정한다.

- 마이페이지 결제·환불 실패 원문과 알 수 없는 enum 비노출
- 웹·Expo 결제 화면의 provider·webhook·API 중심 안내 제거
- 선택과 시술 계획 확정, 재시도와 새 작업 다시 생성의 구분
- 결제 후 자동 실행 금지와 최신 비용 재확인 안내
- 비-에프터케어 핵심 화면의 정적 영문 kicker·상태·CTA와 환경 변수·콜백·멱등·서명 링크 같은 구현 용어 비노출

모델이 생성하는 자유 텍스트는 웹·Expo 공통 한국어 표시 resolver와 실패 시 한국어 fallback으로 로컬 계약을 닫았다. 실제 번역 provider 품질, 에프터케어 별도 범위, 인증된 전체 화면 스크린리더·실기기 문맥 검증은 Phase 13 종료 게이트로 유지한다.
